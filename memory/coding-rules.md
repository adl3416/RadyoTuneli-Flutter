# Coding Rules

- Do not change app code unless the task asks for it.
- Do not rewrite unrelated files.
- Do not delete existing code unless explicitly asked.
- Prefer small, targeted diffs.
- Use `rg` for focused search.
- Read only the files needed for the current task.
- Keep terminal output summaries short; include only the last 40 log lines when logs matter.
- Before large edits, explain the plan briefly.
- Follow existing Flutter, Dart, and platform-specific patterns.
